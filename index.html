你的文章散落在哪裡？

Current State（大多數人的現狀）

• 20 年累積的內容，散落各處
• 專欄文章 → 媒體網站上
• 演講稿 → 各年度資料夾裡
• 專訪報導 → 助理剪報 PDF
• 內部備忘錄 → 公司信箱裡

• 想找某篇三年前的觀點？翻半天
• 出版社邀稿怎麼辦？不知道素材夠不夠

Solution（這份手冊教你的）

• 系統化的知識資產庫
• 專欄文章 → 自動歸類、可搜
• 演講稿 → OCR+摘要+標籤
• 專訪報導 → 全文收錄+交叉引用
• 備忘錄 → 主題串連
• 社群貼文 → 依主題歸檔
• 推薦序/書評 → 脈絡納入
• 想找？→ 跟 AI 說「何時談 ESG」→ 3 秒
• 想出書？→ 完整清單+缺口分析
你花 20 年累積的思想結晶，不應該躺在硬碟裡找不到。AI 不是幫你寫新文章—是幫你把已經寫過的變成可策展的資產。

Phase 0-1：安裝 Claude Desktop

Step 1：申請帳號

前往 https://claude.ai → 註冊帳號（Google / Email）→ 訂閱 Claude Pro（$20/月）← 建議　或 Claude Max（$100/月）← 大量內容處理

Step 2：下載 Claude Desktop

前往 https://claude.ai/download → 選擇你的作業系統（macOS / Windows）→ 安裝 Claude Desktop App → 登入你的帳號

Step 3：啟用 Cowork 模式

打開 Claude Desktop App → 左下角 Settings ⚙ → Features → 啟用「Cowork」（研究預覽版）→ 回到主畫面，切換到 Cowork 模式

Step 4：選擇工作資料夾

Cowork 會詢問「選擇一個資料夾」→ 選擇（或新建）你的知識庫資料夾 → 例如：~/Documents/RoseLibrary/ → AI 現在可以讀寫這個資料夾裡的檔案了
[!] Cowork 在本地 VM 中執行。你選的資料夾 = AI 的工作空間。不選 = AI 沒有檔案系統可用 = 只能聊天。

格式：{來源代碼}-{序號}_{日期}_{簡述}.{副檔名}

INT-008_20200601_商周專訪_AI時代.pdf

來源代碼

• COL = Column（專欄）
• SPE = Speech（演講）
• INT = Interview（專訪）
• BKI = BookIntro（推薦序）
• SOC = Social（社群）

批量處理：讓 AI 幫你加速

Prompt 範本（批量處理）

_inbox/ 裡有 8 篇 PDF，都是我的專欄文章。請幫我逐篇處理：1. OCR 提取全文 2. 辨識日期和媒體 3. 建內容卡片 4. 入庫到 C_Content/Columns/ 5. 更新 ContentIndex。一篇一篇處理，每完成一篇就告訴我進度。不確定的資訊先標記「待確認」，不要猜。

逐篇處理的好處

安全：每篇處理完就存檔，不怕中斷。可校正：隨時修正 AI 的判斷。不遺漏：AI 每篇都建卡片。進度可見：今天處理了 10/200 篇。

Prompt 範本（檔案內容不確定）

_inbox/ 裡有一批七雜八的檔案，我不確定每個是什麼。請幫我：1. 先掃描所有檔案，列出每個檔案的類型和可能內容 2. 列出來給我確認 3. 確認後再開始處理。我可能混了一些非文章類的東西（如合約、發票），那些不需要入知識庫，告訴我讓我自己處理。
17 / 27PHASE 2

檔案累積後的管理：讓 AI 幫你重新整理

Reclassification — 請 AI 重新分類

我的知識庫現在有 150+ 篇文章，但分類可能不夠準確。請幫我：1. 掃描所有 C_Content/ 文章 2. 根據內容重新建議分類 3. 看看需不需要新增主題標籤 4. 找出放錯目錄的文章。先列出分類建議，等我確認後再動手。

Consistency Check — 搬移後的關聯性檢查

我剛把一些文章重新分類了，請幫我檢查：1. ContentIndex 裡記錄的檔案位置，跟實際一致嗎？2. 內容卡片裡的路徑需要更新？3. 有沒有孤立的檔案——存在但 ContentIndex 沒記錄？4. 有沒有 ContentIndex 有記錄但檔案不存在的？列出所有需要修正的地方。
情境說明：處理了 100+ 篇文章之後，你可能發現分類不夠細或有些文章放錯目錄。AI 可以一次幫你掃描所有檔案，根據「內容」而不只是「檔名」重新分類，然後檢查每個改動是否反映在追蹤檔裡。10 分鐘就能抓出所有不一致。

跨主題連結 + 你還沒寫過什麼

交叉分析

Prompt：請幫我找出不同主題之間的交叉連結：1.「數位轉型」和「領導力」有哪些交集文章？2. 同一個觀點是否出現在不同主題的文章中？3. 有沒有兩個主題「應該有交集但你從沒一起談過」？用矩陣圖呈現。

缺口分析

Prompt：根據我的所有內容，請分析：1. 我經常提到但從沒深入展開的觀點 2. 讀者/聽眾可能最常問但我沒正面回答的問題 3. 如果要出一本「數位轉型」的書，我現有素材涵蓋了哪些章節？還缺什麼？這些缺口 = 我可能需要新寫的章節。
核心洞察：交叉分析的價值：發現不同主題間的暗線——「數位轉型會導致組織變革，而組織變革需要領導力」。讀者最喜歡這種「橫向連結」的思考。缺口分析的價值：你不是把 42 篇文章釘成一本書——你要知道「書的完整結構需要什麼」。

關鍵決策

[!] 為什麼「不要幫我改寫」很重要：

做法：讓 AI 直接寫出完整章節
風險：失去你的聲音和風格
結果：讀者覺得「這不像 Rose」

[V] 做法：讓 AI 選取和排列，你自己改寫
風險：無（保留原始風格）
結果：讀者聽到你的聲音

核心原因：出書最危險的陷阱是讓 AI 改寫你的文字。讀者買這本書是因為想聽 Rose 怎麼說——不是想聽 AI 怎麼重述 Rose 的觀點。AI 的角色是「素材策展師」，不是「代筆作家」。

結語：這套方法不只適用於出書。同樣的 Workspace，你也可以用來：策展一個演講系列 / 整理一份企業傳記 / 或者只是單純地「知道我這 20 年到底寫了什麼」。AI 的價值不在於幫你創造新內容——而在於幫你看清已有的內容的價值。
